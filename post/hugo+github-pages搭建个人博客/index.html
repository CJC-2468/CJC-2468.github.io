Github Pages + Hugo 搭建个人博客

  • Tue, Apr 19, 2022
  • One-minute read

这篇文章会简单介绍如何利用Github Pages和Hugo 搭建一个简单的个人博客，不涉及云服务器及域名购买。

为什么用hugo+github pages搭建个人博客

Hugo:一个用Go语言编写的静态网站生成器，几分钟就能上手，用来生成我们需要的静态的个人博客。优点是修改MarkDown文章后，页面会自动呈现出最终显示效果，能极大的提高博客书写效率。

Github pages：提供免费的静态网站托管服务，用来存放用hugo生成的静态网站的资源，它可以给你提供免费的域名如XXXX.github.io，让别人能访问你的静态个人博客。

两者结合可以搭建好看并且免费的个人博客。

搭建过程

环境：macos

核心步骤

1.下载安装hugo和git，并准备一下github相关的库

下载hugo：brew install hugo 测试安装是否成功：hugo version

下载git：brew install git 测试安装是否成功：git --version

自己注册创建一个github账号，然后创建一个GitHub repository，库名为 username.github.io。username 即你的 GitHub 账号 username。我的是下面这样：

2.使用Hugo 创建一个网站模版

创建自己想存放网站的目录，我是在Documents下创建hugo-blog这个目录放我的网站资源。

cd Documents

mkdir hugo-blog

进入hugo-blog目录开始正式搭建：

执行hugo new site XXX-blog ,XXX-blog是你自己设置的，执行完这时候ll一下就能看到刚生成的站点资源，也就是构建这个站点的一些基本资源，还缺少主题，所以下一步就是找主题。

3.找个合适的主题

Hugo Themes: https://themes.gohugo.io 自己去上面找一个喜欢的网站主题

cd XXX-blog ,这时候的路径是这样的：

接着在这个目录下创建themes 目录，一般默认有，如果没有自行创建：mkdir -p themes

截止到目前目录是这样的：

进入themes目录后，下载自己喜欢的主题：git clone https://github.com/lxndrblz/anatole.git

参考所选 Hugo 主题的配置说明，编辑 config.toml 文件。这个文件在xxx-blog目录下面，自己对应个人喜好修改。

4.创建一篇文章

在网站根目录（xxx-blog）下创建文章，hugo new post/first-post.md

5.查看当前网站效果

启动 Hugo server： hugo server -D

使用浏览器打开 http://localhost:1313 预览，如果感觉界面还想再配置一下就继续修改config.toml文件，然后刷新页面。

6.正式生成最终的静态网站资源

在网站根目录（xxx-blog）下，执行 hugo，这时候hugo就会自己根据刚才的配置和相关资源生成我们要的静态网站。Hugo 会将构建的网站内容默认保存至网站根目录的 public/ 文件夹中。

7.将静态网站资源转为git库并和第一步中创建的库做一个关联

将xxx-blog/public转为git库，因为编译好的完整资源都在这里。怎么做呢？

cd xxx-blog/public 生成的 HTML 文件保存在 “public” 目录中，“public” 文件夹会被转换为 Git 库。 git init # 初始化 Git 库。

然后关联一下：

git remote add origin git@github.com:username/username.github.io.git 这里的username就是之前你创建的 github库的那个username。

查看当前修改状态 git status

添加修改过得文件 git add .

提交并指定说明信息 git commit -m "first commit"

提交修改至远程库：git push -u origin master

这里如果提交失败，有两种情况，一个是就是本地库和远程库不一致，另一个是你没有配置过ssh-key，这两个都是git相关的基本常见问题，自己可以google一下，很好解决。

8.最后一步

访问https://username.github.io/即可。

总结

整体搭建过程还是比较简单的，需要你了解一些git相关命令即可，遇到的问题也大多是git相关的常见问题，自己google一下就可以解决。
