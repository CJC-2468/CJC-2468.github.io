Hugo添加自定义页面

  • Sat, Apr 30, 2022
  • One-minute read

这里说的页面不是指 hugo 利用 markdown 文件生成的页面，而是我们自己手动创建的 html 页面。

前置知识

一个用hugo生成的网站包含以下文件和文件夹：

  • config.toml: 配置文件
  • content: 储存网站的所有内容
  • data: 储存数据文件供模板调用
  • layouts: 储存.html模板，该文件夹的优先级高于主题下的/layouts文件夹
  • static: 储存图片,css,js等静态文件，该目录下的文件会直接拷贝到/public，该文件夹的优先级高于主题下的/static文件夹
  • themes: 储存主题
  • public: 执行hugo命令后，储存生成的静态文件

接下来主要用到config.toml，static，public文件及文件夹。

目录环境

流程

1.首先在 content 目录下创建 wikif.md 文件

2.使用 hugo 命令生成博客内容

3.删除刚刚生成的wikif.md文件（因为 hugo 首先获取的是wikif.md 中的内容，只有在这个文件不存在时，才会使用 static 中的内容。）

4.进入public 目录，你可以看到出现了一个wikif 目录，将这个目录移动到 static 目录下，之后修改 index.html 里面的内容为你需要的即可，或者这里直接把自己的html文件覆盖掉这个index.html文件即可，记得改名为index.html。

5.使用命令hugo server -D然后打开链接http://localhost:1313/查看效果即可。

注意

1.如果你需要将该页面添加到导航栏，可以参照配置文件中其他导航项的结构，将其添加进去即可，比如我的 toml 设置方式：

2.public 目录中的内容只会修改，而不会删除。比如你之前创建了一篇名为 hello.md 的文章，使用 hugo 命令生成博客内容后，即便你手动删除了 hello.md，并再次使用 hugo 指令生成博客内容，你会发现你博客中依然会存在 hello 这么一篇文章。如果你想彻底删除它，除了删除 hello.md 外，还需要手动将 public 中对应的文件删除掉才行。所以建议可以先整个删除 public 目录，然后再使用 hugo 命令生成博客内容。
